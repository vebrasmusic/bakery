Bakery vNext

Routing-only, multi-pie/multi-slice local orchestration.

Core Idea

  • Pie and slice are metadata + routing units.
  • Bakery allocates ports and route hosts only.
  • Your runtime scripts start services on allocated ports.
  • Bakery does not orchestrate containers.

Control Plane

setup script -> bakery daemon
  POST /v1/slices
    { pieId, resources:[{key,protocol,expose}] }

bakery -> returns
  {
    sliceId,
    host,
    resources:[
      { key:"r1", allocatedPort:30011, routeUrl:"http://pie-s1.localtest.me:4080" }
    ]
  }

Data Plane Routing

browser -> Host: pie-s1.localtest.me(:4080)
   -> Bakery embedded proxy
   -> 127.0.0.1:allocatedPort

Primary host pattern: <pieSlug>-s<ordinal>.localtest.me

Subdomain resources: <resourceKey>.<pieSlug>-s<ordinal>.localtest.me

Operational Notes

  • Default data dir: ~/.bakery
  • Default DB: ~/.bakery/bakery.db
  • CLI lifecycle: bakery up, bakery status, bakery down
  • Dashboard: run bakery with no subcommand.
